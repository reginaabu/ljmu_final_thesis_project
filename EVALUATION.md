# Track 1 – Evaluation Details

## Gold Evidence Definition

### Definition
A retrieved result is a **hit** if at least one document chunk whose `pubid` (PubMed ID) matches the question's annotated `pubid` appears within the top-k results.

Matching is **exact PMID equality** — no word-overlap threshold, no fuzzy matching.

### Why this is valid
PubMedQA `pqa_labeled` was constructed by human annotators who linked each question to the specific PubMed abstract used as evidence. The `pubid` field in the dataset is the stable PubMed identifier for that article. Using PMID matching grounds evaluation in the original human annotation rather than an approximation.

### Limitations
| Limitation | Impact |
|-----------|--------|
| Single gold document per question | May miss other valid evidence articles not in PubMedQA |
| Chunk-level leniency (any chunk counts) | Overstates performance vs. requiring the exact evidence sentence |
| Closed corpus (1,000 abstracts) | Recall scores are higher than they would be in open-domain retrieval over all of PubMed |
| High natural lexical overlap in PubMedQA | Questions and their gold abstracts share vocabulary by construction, which benefits BM25 disproportionately |

## Dataset

| Property | Value |
|----------|-------|
| Dataset | PubMedQA `pqa_labeled` |
| Source | Hugging Face Datasets Hub |
| Split used | `train` (the only split in `pqa_labeled`) |
| Size | 1,000 question–context pairs |
| Gold field | `pubid` (PubMed ID of the evidence article) |

## Preprocessing

See [`docs/preprocessing.md`](./docs/preprocessing.md) for full details.

| Step | Choice |
|------|--------|
| Lowercasing | Yes — applied to both index and queries |
| Stopwords | Not removed |
| Tokenization | Whitespace split (`.split()`) |
| Chunk size | 400 words |
| Chunk overlap | 50 words (step = 350) |
| Total chunks | 1,009 (from 1,000 records) |

---

## Retrieval Method

- **Algorithm:** BM25Okapi (`rank-bm25==0.2.2`)
- **Index unit:** document chunk (not full abstract)
- **Query processing:** lowercase + whitespace split (matching index tokenization)
- **Top-k evaluated:** k = 5 and k = 10

---

## Evaluation Metrics

| Metric | Formula | Interpretation |
|--------|---------|----------------|
| **Recall@k** | hits / N | Fraction of queries where gold doc appears in top-k |
| **MRR@k** | mean(1/rank) | How high the gold doc ranks on average |
| **nDCG@k** | mean(1/log₂(rank+1)) | Rank-weighted recall (higher rank = higher reward) |

---

## Results

Evaluated on all 1,000 PubMedQA questions with PMID-based gold matching:

| Metric | Score |
|--------|-------|
| Recall@5 | 0.963 |
| Recall@10 | 0.975 |
| MRR@5 | 0.934 |
| MRR@10 | 0.936 |
| nDCG@5 | 0.941 |
| nDCG@10 | 0.945 |

Results are saved to `metrics.json` (generated by `python scripts/run_track1.py`).

## Error Analysis

At Recall@10, approximately 25 queries out of 1,000 fail — the gold article does not appear in any of the top-10 retrieved chunks.

Confirmed failure patterns:
1. **Topic collision via shared procedural vocabulary** — retrieved documents share high-level procedural terms (e.g., "efficacious", "safe", "community setting") but cover entirely different medical procedures.
2. **Figurative/non-literal query language** — query uses metaphorical phrasing that has no lexical bridge to the gold document's academic terminology.
3. **Lexical mismatch** — gold document uses domain-specific or abbreviated terminology not present in the query.

See [`error_analysis.md`](./error_analysis.md) for failure cases with per-case diagnosis.
The script `scripts/run_track1.py` auto-generates this file.
